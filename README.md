# work-shift-calendar

Календар змін, який прибирає ручні підрахунки і за кілька секунд показує головне: коли робота, коли вихідний, де є ручні зміни і що відбувається в інших графіках поруч.

Підходить для особистого користування і для невеликої команди. Працює як вебзастосунок і як Android-застосунок, зберігає все локально і лишається корисним навіть без мережі.

**Поточна версія:** `1.0.46`

## Для чого цей застосунок

Це не просто календар місяця. Він потрібен, щоб:

- не рахувати цикл роботи вручну
- за хвилину бачити свій місяць і навантаження по змінах
- швидко фіксувати важливі дні, нотатки і ручні коригування
- тримати поруч додаткові графіки колег або інших змін
- користуватись одним і тим самим календарем і в браузері, і на телефоні

## Що вміє

- готові графіки `5/5`, `4/4`, `5/2`, `3/3`, `2/2`, щоб стартувати без зайвого налаштування
- власний цикл з будь-якою кількістю робочих та вихідних днів
- календар місяця з підрахунком робочих і вихідних днів
- статистика по періоду, включно з нотатками, святами і навантаженням
- ручна зміна статусу конкретного дня і короткі нотатки на день
- блок `Що нового` після оновлення застосунку
- швидкий перехід між сусідніми днями у модальному вікні дня
- державні свята і погода для вибраного дня
- додаткові графіки у розділі `Графіки` з кольорами, редагуванням і показом прямо в календарі
- світла і темна тема
- офлайн-робота після першого завантаження
- коректна поведінка системної кнопки `Назад` на Android: спочатку закриваються відкриті вікна і меню, а не весь застосунок

## Як запустити локально

```bash
npm install
npm run dev:web
```

Після цього застосунок відкривається локально через dev-сервер.

## Android

Потрібні `Node.js 18+`, `JDK 17` і `Android Studio`.

Перший запуск:

```bash
npm run cap:android
npm run android:open
```

Для швидкої перевірки UI на телефоні без нового APK:

```bash
npm run dev:web
npm run android:live
```

Команди синхронізації Android автоматично переприкладають локальні Gradle-патчі після `Capacitor sync`, щоб Android-проєкт лишався сумісним із поточним стеком збірки.

## Основні команди

| Команда                               | Що робить                                       |
| ------------------------------------- | ----------------------------------------------- |
| `npm run dev:web`                     | запускає локальну веб-версію                    |
| `npm run build:web`                   | оновлює зібрану веб-частину в `www/`            |
| `npm run cap:android`                 | збирає веб-частину і синхронізує Android-проєкт |
| `npm run android:open`                | відкриває Android-проєкт в Android Studio       |
| `npm run android:live`                | запускає live reload на телефоні                |
| `npm run version:set -- x.y.z`        | змінює версію застосунку                        |
| `npm run release:check -- beta x.y.z` | перевіряє beta-реліз перед публікацією          |
| `npm run release:beta -- x.y.z`       | публікує beta-реліз                             |
| `npm run release:stable -- x.y.z`     | публікує stable-реліз                           |

## Оновлення

У застосунку є два канали оновлень:

- `stable` — основний канал для звичайного користування
- `beta` — канал, де нові зміни з'являються раніше

Якщо для вашого каналу виходить нова збірка, застосунок може запропонувати її встановити прямо всередині себе.

## Релізи

У проєкті є `beta` і `stable` канал. Для разового переходу потрібної beta-версії в stable можна заздалегідь виставити дату і час через [`.github/one-time-stable-release.json`](.github/one-time-stable-release.json): scheduler сам опублікує stable-реліз, оновить маніфест і дотисне GitHub Pages, щоб застосунок побачив нову версію без ручного пушу. Час задається в UTC, а запуск GitHub cron може зміститись на кілька хвилин від точного моменту.

## Повне очищення

У `Налаштуваннях` є кнопка `Очистити все`. Вона скидає:

- основний графік
- ручні зміни статусів днів
- нотатки
- графіки з розділу `Графіки`
- тему, кольори і локальний кеш застосунку

Кеш свят при цьому не видаляється.

## Коротко про проєкт

Проєкт зроблений на простому фронтенді без важкого фреймворку і загорнутий в Android через `Capacitor`. Основна веб-частина лежить у корені репозиторію, а `www/` містить зібрану копію, яку використовує мобільна оболонка.
