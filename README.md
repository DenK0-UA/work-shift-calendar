# work-shift-calendar

Застосунок для ведення змінного графіка роботи у вебі та на Android через `Capacitor`.

**Поточна версія:** `1.0.44`

## Що є в застосунку

- готові цикли `5/5`, `4/4`, `5/2`, `3/3`, `2/2`
- власний графік зі своїми робочими та вихідними днями
- календар місяця зі статистикою, ручною зміною статусу дня і нотатками
- додаткові графіки (`Графіки`) з кольорами, редагуванням і показом у календарі
- державні свята, темна/світла тема, погодний блок, офлайн-робота
- на Android системний `back` / `swipe-back` спочатку закриває відкритий шар інтерфейсу, а не одразу виходить із застосунку

## Швидкий старт

```bash
npm install
npm run dev:web
```

Після цього застосунок буде доступний локально через `Vite`.

## Android

Потрібні:

- `Node.js 18+`
- `JDK 17`
- `Android Studio`

Перший запуск:

```bash
npm run cap:android
npm run android:open
```

Для швидких UI-перевірок на телефоні без нового APK:

```bash
npm run dev:web
npm run android:live
```

## Основні команди

| Команда                               | Що робить                               |
| ------------------------------------- | --------------------------------------- |
| `npm run dev:web`                     | локальний веб-режим розробки            |
| `npm run build:web`                   | збирає веб-частину в `www/`             |
| `npm run cap:android`                 | збирає веб і синхронізує Android-проєкт |
| `npm run android:open`                | відкриває `android/` в Android Studio   |
| `npm run android:live`                | запускає live-reload на телефоні        |
| `npm run version:set -- x.y.z`        | оновлює версію застосунку               |
| `npm run release:check -- beta x.y.z` | preflight перед beta-релізом            |
| `npm run release:beta -- x.y.z`       | публікує beta-збірку                    |
| `npm run release:stable -- x.y.z`     | публікує stable-збірку                  |

## Оновлення

- `beta` — тестовий канал
- `stable` — основний канал
- APK публікуються в `GitHub Releases`, а маніфести оновлень — через `GitHub Pages`

## Скидання даних

Кнопка `Повне очищення` у вікні `Вигляд` видаляє:

- графік і кастомні статуси
- нотатки
- додаткові графіки з `Графіки`
- тему, кольори й локальний кеш застосунку

> Кеш свят не видаляється.

## Стек

`HTML` + `CSS` + `Vanilla JavaScript` + `Capacitor Android`
