# work-shift-calendar

Статичний веб-застосунок для ведення змінного графіка, перегляду календаря, ручного коригування статусів днів і збереження нотаток.

## Що вміє

- Підтримує готові цикли `5/5`, `4/4`, `5/2`, `3/3`, `2/2`
- Дозволяє налаштувати власний кастомний графік
- Працює зі стартовою датою циклу
- Показує календар, статистику й деталі дня
- Дозволяє вручну змінювати статус окремого дня
- Зберігає нотатки для конкретних дат
- Показує державні свята
- Підтягує прогноз погоди для вибраного дня через `Open-Meteo`
- Зберігає дані та налаштування у `localStorage`

## Поточна поведінка

- Для нових користувачів першим відкривається екран вибору графіка
- Поки графік не вибраний, основний календар не показується
- Розділ `Вигляд 🎨` відповідає за тему, кольори й візуальні дії
- Режим теми `Авто` бере поточну тему пристрою через `prefers-color-scheme`
- Публічно активним лишається стиль `Current`

## Вигляд і скидання

У вікні `Вигляд 🎨` доступно:

- перемикання режиму теми: `Авто`, `Світла`, `Темна`
- налаштування кольору робочих і вихідних днів
- `Скинути вигляд`, що скидає лише тему, кольори й візуальні параметри
- повний reset у danger-zone, який запускається утриманням кнопки протягом `2` секунд і після підтвердження очищає всі локальні дані застосунку

## Запуск

Проєкт не потребує збірки.

```bash
start index.html
```

Альтернативно:

- відкрити `index.html` напряму в браузері
- запустити через `Live Server` у редакторі

## GitHub Pages

У репозиторії є workflow для публікації через `GitHub Pages` з гілки `main`.

- після push у `main` сайт оновлюється автоматично
- у налаштуваннях репозиторію для `Pages` має бути вибране джерело `GitHub Actions`

## Як користуватись

1. Для першого запуску виберіть свій графік і старт циклу.
2. Переглядайте місяці кнопками навігації або жестами на мобільних.
3. Натискайте на день, щоб відкрити деталі, змінити статус або додати нотатку.
4. Відкрийте `Ваш Графік 📅`, щоб змінити цикл або дату старту.
5. Відкрийте `Вигляд 🎨`, щоб змінити тему й кольори.

## Примітки

- для оновлення погоди потрібен доступ до `api.open-meteo.com`
- без інтернету календар і локальні дані працюють далі, але прогноз не оновлюється
- застосунок побудований на `HTML`, `CSS` і `Vanilla JavaScript`
