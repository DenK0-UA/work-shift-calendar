# Work Shift Calendar

Календар змін для тих, хто працює позмінно і не хоче тримати графік у голові. Застосунок показує робочі та вихідні дні, ручні зміни, нотатки, свята, погоду і додаткові графіки поруч з основним календарем.

Працює у браузері та як Android-застосунок на Capacitor. Дані користувача зберігаються локально, а основний інтерфейс лишається доступним після першого завантаження.

## Що вміє застосунок

- календар місяця з підрахунком робочих і вихідних днів;
- готові графіки `5/5`, `4/4`, `5/2`, `3/3`, `2/2`;
- власний цикл з окремою кількістю робочих і вихідних днів та датою старту;
- ручна зміна статусу конкретного дня з можливістю скасування;
- короткі нотатки на день з лічильником символів;
- статистика за місяць, 3 місяці або рік: робочі, вихідні, серії, ручні зміни, нотатки, свята і навантаження;
- державні свята України з кешуванням;
- прогноз погоди для вибраного дня;
- додаткові графіки у розділі `Графіки`: назва, колір, видимість, редагування, готовий або власний цикл;
- показ активних додаткових графіків у календарі та в модальному вікні дня;
- блок `Що нового` після оновлення застосунку;
- швидкий перехід між сусідніми днями у модальному вікні дня, включно зі свайпом;
- теми `Авто`, `Світла`, `Темна`;
- власні кольори робочих і вихідних днів;
- офлайн-робота основного інтерфейсу після першого завантаження;
- коректна поведінка системної кнопки `Назад` на Android: спочатку закриваються відкриті вікна, меню і фільтри.

## Оновлення

У застосунку є два канали оновлень:

- `stable` - основний канал для щоденного користування;
- `beta` - тестовий канал, де зміни з'являються раніше.

Поточні маніфести оновлень:

- `stable/version.json` - стабільна версія `1.0.52`;
- `beta/version.json` - тестова версія `1.0.52`.

В Android-застосунку перевірка оновлень читає маніфести з GitHub Pages. Якщо доступна нова версія, застосунок відкриває офіційну сторінку GitHub Release. На сторінці релізу потрібно завантажувати саме APK-файл у секції `Assets`; файли `Source code` GitHub додає автоматично і для встановлення застосунку вони не потрібні.

Поточна версія `1.0.52` не встановлює APK повністю всередині застосунку. Актуальний сценарій - відкрити сторінку релізу і вибрати APK вручну.

`beta`-канал прихований для звичайного користувача і керується через `beta/access.json` або локальне ввімкнення на конкретній інсталяції.

## Дані та очищення

Основний графік, ручні зміни, нотатки, додаткові графіки, тема, кольори й стан оновлень зберігаються локально у браузері або Android WebView.

Погода завантажується через Open-Meteo. Свята завантажуються через Nager.Date і кешуються локально.

У `Налаштуваннях` є кнопка `Очистити все`. Вона видаляє основний графік, ручні зміни, нотатки, додаткові графіки, тему, кольори, стан оновлень, browser cache і service worker. Локальний кеш свят у `localStorage` не видаляється.

## Структура проєкту

- `index.html` - основна HTML-оболонка застосунку;
- `js/` - логіка календаря, графіків, оновлень, налаштувань, погоди і Android Back;
- `styles/` - стилі інтерфейсу та адаптивність;
- `data/config.js` - версія застосунку, URL маніфестів, release notes, свята і базові налаштування;
- `stable/version.json` і `beta/version.json` - публічні маніфести оновлень;
- `beta/access.json` - список дозволених beta-інсталяцій;
- `android/` - Capacitor Android-проєкт і нативний APK download helper;
- `scripts/` - збірка, версіонування, preflight і release helpers;
- `.github/workflows/` - GitHub Actions для Pages, beta APK і stable promotion.

## Дозволена розробка

Ці команди наведені для власника проєкту або розробників, які мають письмовий дозвіл. Вони не надають ліцензію на зміну, збірку, розповсюдження або створення похідного продукту.

```bash
npm ci
npm run dev:web
npm run build:web
npm run lint
```

Android-синхронізація та відкриття Android Studio:

```bash
npm run cap:android
npm run android:open
```

Оновлення версії застосунку:

```bash
npm run app:version:set -- 1.0.53
```

Release preflight:

```bash
npm run release:check -- beta 1.0.53
npm run release:check -- stable 1.0.53
```

Публікація релізів виконується з чистого `main` через теги, які пушать npm-скрипти:

```bash
npm run release:beta -- 1.0.53
npm run release:stable -- 1.0.53
```

Beta workflow збирає підписаний APK і оновлює `beta/version.json`. Stable workflow бере APK з відповідного beta-релізу, публікує stable release і оновлює `stable/version.json`.

## Коротко

Work Shift Calendar - це практичний календар для щоденного користування: відкрив, побачив зміну, перевірив сусідні графіки, додав нотатку або ручну зміну і пішов далі.
